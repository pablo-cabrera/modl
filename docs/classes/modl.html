unit

(
  • payload
)
static

Defined in lib/modl.js:1994

Creates a Loader and schedules a given payload function export the asset. The payload will only run when all the requires are loaded and ready for prime time.

When called, the payload function will receive the module struct as its first argument. The imports will be passed as subsequent arguments in order.

The module object will have an exports property which will be published for this asset. This property can be overwritten with any value this asset may want to publish (a la nodejs module system).

The module object will also have an imports object containing all required resources.

Every imports will be passed along as parameters to the payload function

Parameters:

  • payload Function

    The payload function

uses

(
  • resource
  • [alias]
)
modl.Loader static chainable

Defined in lib/modl.js:1962

Creates a Loader and requires an asset from it. The resource string may have the following formats:

  • "/path/to/asset: in this case the asset within the current module will be loaded

  • "module": in this case the "module" within node_modules folder will be required, and afterwards the /module asset within the module will be loaded.

Optionally an alias property may be passed to name the required asset within the imports object. If ommited, the alias will be automatically generated based the resource value.

Parameters:

  • resource String

    A given resource to be required

  • [alias] String optional

    An optional alias which to be named within the imports object.
